# FirstCircle User Flows

This document outlines the step-by-step user pathways through the FirstCircle platform.

---

## 1. Registration & Profile Setup
1.  **Sign Up / Login**: User creates an account or logs in.
2.  **Basic Info**: Sets name, bio, age, gender, and uploads a profile picture.
3.  **Interest Chips**: Selects 5-10 interest keywords (e.g., `#tech`, `#boardgames`, `#coffee`, `#indierock`).
4.  **Comfort Chips**: Defines preferences regarding group size (3, 4, 5, or 6), activity expense level, and conversational styles.
5.  **Free Slot Form**: Fills in weekly calendar slots (e.g., Friday 6 PM - 10 PM, Saturday 12 PM - 4 PM) representing free windows.

---

## 2. Activity Discovery & "Drops"
1.  **Browse Drops**: User navigates the active list of Drops, filtering by category, distance, and time.
2.  **Join Drop**: User joins a Drop and submits a "Vibe Vote" (e.g., Chill vs. Active).
3.  **Create Drop**: User drafts a title, descriptions, selects coordinates/location, sets date/time, and sets minimum group size. The creator is the default host.

---

## 3. Blind Proposal Phase
1.  **Matching Engine Run**: Periodically, the matching engine runs. It groups users who joined a Drop (or via Auto-Place).
2.  **Blind Proposal Alert**: Users receive a notification of a matching Circle proposal.
3.  **Proposal Review**:
    *   The proposal page shows *blind profiles*: ages, genders, reliability stats, and shared interests (e.g., "3 members share `#boardgames`"), but hides names and profile pictures.
    *   A radial countdown timer (e.g., 2 hours) is displayed.
4.  **Accept or Skip**:
    *   **Accept**: User votes yes.
    *   **Skip**: User rejects. No penalty for skipping, but the group is dissolved.

---

## 4. Confirmed Circle & Rescheduling
1.  **Reveal**: If all proposed members accept before the countdown expires, the Circle is formed.
2.  **Full Reveal**: Real names, chat room, and exact location are shown.
3.  **Chatting**: Members coordinate travel.
4.  **Reschedule Control**: If a conflict arises, members can request a reschedule. If all agree, the system locks a new time slot.

---

## 5. Post-Event Feedback Loops
1.  **Feedback Prompt**: 2 hours after the event, a notification prompts a rating.
2.  **Feedback Form**:
    *   Confirm attendance of each other member (Did they show up? On time or late?).
    *   Provide vibe reviews (e.g., friendly, quiet, overbearing).
    *   Optional safety/harassment reports.
3.  **Reliability Update**: System updates reliability scores based on reports.
